# 粤通卡ETC会员营销平台 - 系统交互说明

---

## 一、系统清单

| 系统 | 性质 | 职责 |
|------|------|------|
| **营销系统** | 内部 | 会员全生命周期管理（产品、权益、订单、扣费调度） |
| **互联网系统** | 内部 | ETC申办发行、签约管理（通行费签约+会员签约） |
| **综合服务系统** | 内部 | 只换不修（处理维保期） |
| **ETC记账系统** | 内部 | 通行费记账扣费，执行扣费指令，管理会员信息 |
| **签约渠道** | 外部 | 支付宝/微信/银联，提供签约和扣费能力 |
| **粤运系统** | 外部 | 权益服务提供方，开通会员免费期 |

## 三、交互流程

### 3.1 ETC申办

```
用户 → 互联网系统 → 签约渠道

① 用户在互联网系统发起ETC申办
② 互联网系统调用签约渠道完成通行费签约
③ 签约渠道返回签约结果 → 互联网系统
④ ETC申办成功，等待卡签寄达
```

### 3.2 ETC激活 + 会员购买

```
用户 → 互联网系统 → 营销系统 → 互联网系统 → 签约渠道 → 综合服务系统/ETC记账系统/粤运系统

① 用户收到ETC卡签，在互联网系统进行激活
② 互联网系统通知营销系统：ETC激活成功
③ 营销系统展示激活成功页 + 会员产品推荐
④ 用户选择会员产品 → 同意协议
⑤ 互联网系统调用营销系统发起会员签约
⑥ 营销系统调用互联网系统完成会员签约
⑦ 互联网系统调用签约渠道完成会员签约
⑧ 签约渠道返回签约结果 → 互联网系统 → 营销系统
⑨ 签约成功后进入审核流程，审核通过后继续
   - 审核失败：订单取消，通知用户
⑩ 营销系统发送扣费指令给ETC记账系统
   - 年卡：一次性扣费¥138
   - 月卡：扣首月¥16.8
   - 次卡：不扣费
⑩ ETC记账系统通过签约渠道执行扣费
⑪ 扣费结果返回 → ETC记账系统 → 营销系统
⑫ 扣费成功 → 营销系统创建会员（TRIAL，61天免费期）→ 分配权益
⑬ 营销系统通知综合服务系统：只换不修（维保期处理）
⑭ 营销系统通知ETC记账系统：会员信息
⑮ 营销系统通知粤运系统：开通会员免费期
```

### 3.3 免费期到期扣费

```
营销系统 → ETC记账系统 → 签约渠道 → 粤运系统

① 营销系统到期前3天/1天发送提醒通知
② 免费期结束 → 营销系统发送扣费指令给ETC记账系统
   - 年卡：一次性扣费¥138（续费一年）
   - 月卡：扣¥16.8（续费一个月）
   - 次卡：无此环节（按次扣费，无免费期概念）
③ ETC记账系统通过签约渠道执行扣费
④ 扣费结果返回 → ETC记账系统 → 营销系统
⑤ 扣费成功 → 会员状态TRIAL→ACTIVE，取消免费期权益，分配正式权益
⑥ 营销系统通知粤运系统：免费期结束，会员转正式
⑦ 扣费失败 → 记录失败日志，每日轮扣重试
⑧ 连续3天扣费失败 → 自动取消会员
⑨ 营销系统通知粤运系统（BILLING_FAILED_CANCEL）
⑩ 营销系统通知ETC记账系统（会员取消）
```

### 3.4 次卡通行扣费

```
ETC记账系统 → 营销系统 → ETC记账系统 → 签约渠道

① ETC记账系统发送通行扣费指令给营销系统
② 营销系统校验会员状态、检查月封顶
③ 营销系统计算扣费金额，返回扣费指令给ETC记账系统
④ ETC记账系统通过签约渠道执行扣费
⑤ 扣费结果返回 → ETC记账系统
```

### 3.5 会员取消

```
用户/管理员/扣费引擎 → 营销系统 → 粤运系统/ETC记账系统/互联网系统/综合服务系统

① 营销系统更新会员状态为PENDING_CANCEL
② 营销系统通知粤运系统（MEMBER_CANCELLED）
③ 营销系统通知ETC记账系统（会员取消）
④ 营销系统通知综合服务系统（取消只换不修权益）
⑤ 到期日 → 营销系统通知互联网系统解除会员签约
⑥ 到期日 → 营销系统取消权益
⑦ 会员状态变为CANCELLED
```

### 3.6 权益核销

```
粤运系统 → 营销系统

① 粤运系统通知营销系统：粤运拯救服务已使用
② 营销系统核销对应权益（usedCount+1）
```

### 3.7 ETC设备注销

```
互联网系统 → 营销系统 → 粤运系统/ETC记账系统/综合服务系统

① 互联网系统通知营销系统：ETC设备注销
② 营销系统自动取消会员（cancelReason: ETC_CANCELLED）
③ 营销系统通知粤运系统
④ 营销系统通知ETC记账系统
⑤ 营销系统通知综合服务系统（取消只换不修权益）
```

### 3.8 只换不修（维保期处理）

```
营销系统 → 综合服务系统

① 会员开通时，营销系统通知综合服务系统：该用户享有只换不修权益
② 综合服务系统记录维保期信息
③ 用户申请ETC设备更换时，综合服务系统校验维保期
④ 维保期内：免费更换设备
⑤ 维保期外：用户自行承担费用
⑥ 会员取消时，营销系统通知综合服务系统：取消只换不修权益
```

## 四、接口清单

### 4.1 营销系统 ↔ 互联网系统

| 接口 | 方向 | 说明 | 触发时机 |
|------|------|------|---------|
| 会员签约请求 | 营销→互联网 | 发起会员签约 | 用户购买会员时 |
| 解约请求 | 营销→互联网 | 解除会员签约 | 会员取消到期时 |
| 签约结果回调 | 互联网→营销 | 签约成功/失败 | 签约渠道回调后 |
| ETC激活通知 | 互联网→营销 | ETC设备已激活 | 用户激活ETC卡签 |
| ETC注销通知 | 互联网→营销 | ETC设备已注销 | 用户注销ETC设备 |

### 4.2 营销系统 ↔ ETC记账系统

| 接口 | 方向 | 说明 | 触发时机 |
|------|------|------|---------|
| 扣费指令 | 营销→记账 | 会员签约扣费/免费期到期扣费/月卡月扣 | 各扣费场景 |
| 扣费结果 | 记账→营销 | 扣费成功/失败 | 扣费执行后 |
| 通行扣费指令 | 记账→营销 | 次卡用户通行高速 | ETC通行时 |
| 会员信息通知 | 营销→记账 | 会员开通/取消信息 | 会员状态变更时 |

### 4.3 营销系统 ↔ 粤运系统

| 接口 | 方向 | 说明 | 触发时机 |
|------|------|------|---------|
| 开通免费期通知 | 营销→粤运 | 会员开通，进入免费期 | 会员签约扣费成功后 |
| 免费期结束通知 | 营销→粤运 | 免费期结束，会员转正式 | 扣费成功TRIAL→ACTIVE |
| 会员取消通知 | 营销→粤运 | MEMBER_CANCELLED | 会员取消后 |
| 扣费失败取消通知 | 营销→粤运 | BILLING_FAILED_CANCEL | 连续3天扣费失败后 |
| 权益核销通知 | 粤运→营销 | 粤运拯救服务已使用 | 粤运提供救援服务后 |

### 4.4 营销系统 ↔ 综合服务系统

| 接口 | 方向 | 说明 | 触发时机 |
|------|------|------|---------|
| 只换不修开通通知 | 营销→综合 | 会员开通，享有只换不修权益 | 会员签约扣费成功后 |
| 只换不修取消通知 | 营销→综合 | 会员取消，取消只换不修权益 | 会员取消/注销后 |
| 维保期查询 | 营销→综合 | 查询ETC设备维保到期时间 | 会员开通时 |

### 4.5 互联网系统 ↔ 签约渠道

| 接口 | 方向 | 说明 | 触发时机 |
|------|------|------|---------|
| 通行费签约 | 互联网→渠道 | 完成ETC通行费签约 | ETC申办时 |
| 会员签约 | 互联网→渠道 | 完成会员产品签约 | 用户购买会员时 |
| 扣费 | 互联网→渠道 | 从签约渠道扣费 | ETC记账系统执行扣费时 |
| 退款 | 互联网→渠道 | 退还扣费 | 订单退款时 |
| 解约 | 互联网→渠道 | 解除签约 | 会员取消到期时 |
| 签约/扣费结果回调 | 渠道→互联网 | 成功/失败 | 签约/扣费完成后 |

## 五、数据流向总结

```
互联网系统（内部）── 顶层
ETC申办发行、签约管理
├──→ 签约渠道（外部）：申办签约时调用
└──→ 营销系统（内部）：ETC激活/注销事件
         │
         ├──→ 综合服务系统（内部）：只换不修/维保期
         ├──→ ETC记账系统（内部）：会员信息/扣费
         └──→ 粤运系统（外部）：开通免费期/权益服务
```

**互联网系统是顶层系统**，ETC申办签约时才调用签约渠道：
- **签约渠道**：仅在申办签约时由互联网系统调用
- **营销系统**：接收互联网系统的ETC激活/注销事件，管理会员全生命周期
- **综合服务系统**：只换不修（维保期处理）
- **ETC记账系统**：执行扣费、通行记账
- **粤运系统**：权益服务提供方
