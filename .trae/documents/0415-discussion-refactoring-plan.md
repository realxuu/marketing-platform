# 营销平台改造计划 — 基于0415讨论内容

## 背景

对照飞书「营销平台系统方案」V1.5（0415讨论更新）与当前代码现状，梳理需要改造的点。

## 回退策略

所有改动基于 Git 管理，改造前先提交当前代码作为基线，任何改动都可安全回退。

---

## 改造点总览

### 一、数据模型改造（Prisma Schema）

#### 1.1 User 模型 — 增加"跟车走"支持
- **现状**：User 仅有 `plateNumber`（单字段），不支持多车牌、车牌颜色
- **改造**：
  - 新增 `plateColor` 字段（车牌颜色：蓝/黄/绿）
  - 新增 `idNumber` 字段（证件号码，同步给粤运需要）
  - 新增 Vehicle 模型（一用户多车牌场景）

#### 1.2 MemberProduct 模型 — 去掉有效期，增加生效起始时间
- **现状**：有 `duration`（有效期天数）和 `isActive`
- **改造**：
  - 去掉 `duration` 字段
  - 新增 `effectiveStartTime`（生效起始时间，DateTime?）
  - 会员类型去掉季卡（`type` 仅保留 YEARLY / MONTHLY / PER_USE）

#### 1.3 Right 模型 — 增加总量控制和详情HTML
- **现状**：有 `type`（SELF_OPERATED / COMMISSION）和 `isActive`
- **改造**：
  - 去掉 `type` 字段（无需配置代销/自营）
  - 新增 `totalLimit`（总量控制，Int?）
  - 新增 `currentTotal`（已发放总量，Int @default(0)）
  - 新增 `detailHtml`（权益详情展示HTML，点击权益详情时展示的内容，String?）

#### 1.4 Member 模型 — 增加取消场景和维保期
- **现状**：`status` 仅有 TRIAL / ACTIVE / EXPIRED / CANCELLED
- **改造**：
  - `status` 增加 `PENDING_CANCEL`（待取消，权益保留至到期）
  - 新增 `cancelReason`（取消原因：USER_CANCEL / ADMIN_CANCEL / BILLING_FAILED / ETC_CANCELLED）
  - 新增 `cancelAt`（取消时间）
  - 新增 `warrantyEndDate`（维保到期日期）
  - 新增 `plateNumber`（绑定车牌，支持"跟车走"）
  - 新增 `plateColor`（车牌颜色）

#### 1.5 Order 模型 — 支持购销分离
- **现状**：订单直接关联 userId + productId，支付即创建会员
- **改造**：
  - 新增 `channel`（签约渠道：WECHAT / ALIPAY / UNIONPAY）
  - 新增 `isActivated`（是否已激活，Boolean @default(false)）
  - 新增 `activatedAt`（激活时间，DateTime?）
  - 新增 `agreementAccepted`（用户协议是否已确认，Boolean @default(false)）
  - `status` 增加 `PENDING_ACTIVATION`（待激活）

#### 1.6 UserRight 模型 — 权益禁用后保留逻辑
- **现状**：权益禁用后无特殊处理
- **改造**：
  - 新增 `rightDisabledAt`（权益被禁用的时间，用于判断旧会员保留权益至会员期结束）

#### 1.7 新增 Vehicle 模型
- **用途**：支持一用户多车牌、车牌切换
- **字段**：id, userId, plateNumber, plateColor, isPrimary, createdAt

### 二、购买流程改造（购销分离模式）

#### 2.1 购买阶段改造
- **现状**：`/purchase` 页面直接购买 → 模拟支付 → 立即创建会员
- **改造**：
  - 购买页面增加"签约渠道选择"（微信/支付宝/银联）
  - 判断渠道是否支持售卖会员权益
  - 支付宝需2次签约提示
  - 购买仅创建订单（不创建会员），订单状态为 `PENDING_ACTIVATION`
  - 增加"用户协议"弹窗确认（运营中心提供内容）
  - 去掉季卡选项

#### 2.2 激活阶段新增
- **现状**：无激活流程
- **改造**：
  - 新增 `/activate` 页面（ETC激活后回调）
  - 激活时完成签约代扣
  - 绑定车辆与会员资格
  - 触发白名单 → 开通免费体验期
  - 通知粤运会员开通信息

#### 2.3 API 改造
- `POST /api/orders`：不再自动创建 Member，仅创建待激活订单
- 新增 `POST /api/activate`：激活接口，完成代扣签约 + 创建会员 + 发放权益
- 新增 `POST /api/notify/yueyun`：通知粤运会员状态变更

---

### 三、会员取消流程改造

#### 3.1 取消场景实现
- **现状**：前端仅 `alert('取消成功（演示）')`，后端无取消API
- **改造**：实现4种取消场景
  1. **用户手工取消**：`/member` 页面取消按钮 → 弹窗提示阅读 → 调用API
  2. **后台系统取消**：`/admin/members` 页面增加"取消会员"操作
  3. **扣费失败取消**：扣费引擎连续3天扣费失败 → 自动取消
  4. **注销ETC取消**：需判断是否绑定会员，已绑定需提示（提示语待确认）

#### 3.2 取消后处理逻辑
- 取消后通知粤运（不进行退款）
- 权益保障至会员到期
- 只换不修服务延续至会员期结束
- 到期后维保期恢复为产品发行时记录的维保期
- 会员状态先变为 `PENDING_CANCEL`，到期后变为 `CANCELLED`

#### 3.3 API 改造
- 新增 `PATCH /api/members/cancel`：取消会员（含取消原因）
- 扣费引擎增加连续失败检测逻辑

### 四、权益管理改造

#### 4.1 权益配置改造
- **现状**：权益有 `type`（自营/代销）字段
- **改造**：
  - 去掉代销/自营分类（无需配置）
  - 增加总量控制配置（totalLimit + currentTotal）
  - 增加权益详情HTML（点击权益详情时展示的内容）
  - 权益禁用时：旧会员保留权益至会员期结束

#### 4.2 产品管理改造
- **现状**：产品有 `duration`（有效期天数）
- **改造**：
  - 去掉有效期字段
  - 增加生效起始时间
  - 扣会员费时判断权益和产品状态

#### 4.3 API 改造
- `PATCH /api/rights`：增加总量控制、详情HTML字段
- `PATCH /api/products`：增加生效起始时间，去掉有效期
- 权益禁用时增加旧会员保留逻辑
