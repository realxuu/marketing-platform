## 一、项目概述

### 1.1 项目背景

粤通卡ETC会员营销平台，为ETC用户提供会员服务（年卡/月卡/次卡）。用户在ETC申办时可体验会员，收到ETC产品激活后自动扣费开通。平台与粤运系统对接，实现权益服务的闭环管理，通过会员机制实现收入多元化。

### 1.2 项目目标

- 搭建完整业务闭环，覆盖会员全生命周期
- 支持多种付费模式（年卡/月卡/次卡）
- 整合权益服务，对接粤运系统
- 现代SaaS风格界面，适合演示和运营

### 1.3 范围界定

**项目范围（In Scope）**

| 类别 | 功能 |
|------|------|
| 用户端 | 会员购买、权益查看、扣费记录、取消退款 |
| 后台端 | 仪表盘、产品管理、权益管理、订单管理、会员管理、扣费控制、结算对账、系统配置 |
| 外部对接 | 粤运系统、ETC记账系统、互联网系统、综合服务系统 |

**项目范围外（Out of Scope）**

- 权益包组合购买
- 权益转赠功能

## 三、核心业务规则

### 3.1 二次签约机制

用户需要完成两次独立的签约：

| 签约类型 | 触发时机 | 签约内容 | 执行系统 |
|---------|---------|---------|---------|
| **通行费签约** | ETC申办时 | 授权从签约渠道扣取ETC通行费 | 互联网系统 |
| **会员签约** | 购买会员时 | 授权从签约渠道扣取会员费 | 互联网系统 |

**签约渠道选项**：支付宝、微信、银联云闪付

### 3.2 激活扣费规则

**核心原则：申办时不扣费，ETC激活时才扣费。**

```
ETC申办 ──→ 创建订单（不扣费）──→ 收到ETC产品 ──→ 激活 ──→ 扣费开通会员
```

**扣费金额规则**：

| 产品类型 | 激活时扣费 | 后续扣费 |
|---------|-----------|---------|
| 年卡 | ¥138（一次性） | 体验期结束后续费¥138/年 |
| 月卡 | ¥16.8（首月） | 每月¥16.8 |
| 次卡 | ¥0（不扣费） | 每次通行¥1，月封顶¥20 |

### 3.3 一车一会员

**规则**：每辆车只能购买一个会员产品。

**校验时机**：ETC激活时、会员购买时

**校验逻辑**：
- 检查车牌号是否已有生效中的会员（状态为 TRIAL、ACTIVE、PENDING_CANCEL）
- 有则拒绝，提示"该车牌已有生效中的会员"

### 3.4 免费体验期

**时长**：61天

**规则**：
- 新会员开通后状态为 TRIAL（体验期）
- 体验期内可随时取消，不产生费用
- 体验期结束前3天、1天发送提醒通知
- 体验期满未取消，自动扣费转正（状态变为 ACTIVE）

**注意事项**：
- 次卡无免费体验期概念
- 取消后再次开通不享受免费体验期

### 3.5 会员取消规则

**取消来源**：

| 来源 | 代码 | 说明 |
|------|------|------|
| 用户主动取消 | USER_CANCEL | 用户在前端操作，需5秒倒计时确认 |
| 管理员取消 | ADMIN_CANCEL | 后台管理员操作 |
| 扣费失败取消 | BILLING_FAILED | 连续3天扣费失败自动取消 |
| ETC注销取消 | ETC_CANCELLED | ETC设备注销时自动取消 |

**取消流程**：
- USER_CANCEL、ADMIN_CANCEL、BILLING_FAILED：状态先变为 PENDING_CANCEL，权益保留至到期日，到期后正式取消
- ETC_CANCELLED：立即取消，无过渡期

#### 年卡会员（YEARLY）

| 属性 | 值 |
|------|-----|
| 价格 | ¥138/年 |
| 扣费方式 | 一次性扣费 |
| 有效期 | 12个月 |
| 体验期 | 61天 |
| 包含权益 | 粤运拯救（不限金额）、只换不修、中石化年卡权益 |

#### 月卡会员（MONTHLY）

| 属性 | 值 |
|------|-----|
| 价格 | ¥16.8/月 |
| 扣费方式 | 每月扣费 |
| 有效期 | 1个月（自动续费） |
| 体验期 | 61天 |
| 包含权益 | 粤运拯救（单次限额500元） |

#### 次卡会员（PER_USE）

| 属性 | 值 |
|------|-----|
| 价格 | ¥1/次 |
| 扣费方式 | 每次通行扣费 |

## 四、业务流程

### 4.1 ETC申办流程

**前置条件**：用户未办理ETC

```
┌─────────┐     ┌──────────────┐     ┌──────────────┐
│ 用户发起  │────→│ 互联网系统    │────→│ 签约渠道      │
│ ETC申办  │     │ 处理申办      │     │ 通行费签约    │
└─────────┘     └──────────────┘     └──────────────┘
                       │                    │
                       │    签约成功 ◄───────┘
                       ▼
                ┌──────────────┐
                │ ETC申办成功   │
                │ 等待卡签寄达  │
                └──────────────┘
```

**流程步骤**：

| 步骤 | 执行方 | 说明 |
|------|--------|------|
| 1 | 用户 | 在互联网系统发起ETC申办 |
| 2 | 互联网系统 | 选择签约渠道（支付宝/微信/银联） |
| 3 | 互联网系统 | 调用签约渠道完成通行费签约 |
| 4 | 签约渠道 | 返回签约结果 |
| 5 | 互联网系统 | ETC申办成功，等待卡签寄达 |

```
                                               │
                                               ▼

                    │              用户选择会员产品                  │
                    │              同意协议，确认购买                │
                    └─────────────────────────────────────────────┘

                                               ▼
┌─────────────┐     ┌──────────────┐     ┌──────────────┐

                    │              签约成功，进入审核                │

                                               │
                                               ▼
┌─────────────┐     ┌──────────────┐     ┌──────────────┐

│ 发送扣费指令│     │ 执行扣费      │     │ 从账户扣费    │

                    ┌─────────────────────────────────────────────┐
                    │              扣费成功                         │
                    │              创建会员（TRIAL，61天体验期）       │
                    │              分配权益                          │
                    └─────────────────────────────────────────────┘
                                               │
                    ┌──────────────────┼──────────────────┐
                    ▼                  ▼                  ▼
            ┌──────────────┐   ┌──────────────┐   ┌──────────────┐
            │综合服务系统   │   │ETC记账系统    │   │ 粤运系统      │
            │只换不修开通  │   │会员信息同步   │   │ 免费期开通    │
            └──────────────┘   └──────────────┘   └──────────────┘
```

**流程步骤**：

| 步骤 | 执行方 | 操作 | 说明 |
|------|--------|------|------|
| 1 | 用户 | 激活ETC | 在互联网系统完成激活 |
| 2 | 互联网系统 | 通知营销系统 | 发送ETC激活通知 |
| 3 | 营销系统 | 展示会员产品 | 显示可选的会员套餐 |
| 4 | 用户 | 选择产品 | 选择年卡/月卡/次卡 |
| 5 | 用户 | 同意协议 | 勾选会员服务协议 |
| 6 | 营销系统 | 发起会员签约 | 调用互联网系统 |
| 7 | 互联网系统 | 调用签约渠道 | 完成会员签约 |
| 8 | 签约渠道 | 返回结果 | 签约成功/失败 |
| 9 | 营销系统 | 审核流程 | 审核通过继续，审核失败则订单取消 |
| 10 | 营销系统 | 发送扣费指令 | 调用ETC记账系统 |
| 11 | ETC记账系统 | 执行扣费 | 通过签约渠道扣费 |
| 12 | 签约渠道 | 返回扣费结果 | 成功/失败 |
| 13 | 营销系统 | 创建会员 | 状态TRIAL，61天体验期 |
| 14 | 营销系统 | 分配权益 | 检查总量限制 |
| 15 | 营销系统 | 多方通知 | 综合服务系统、ETC记账系统、粤运系统 |

**订单状态流转**：
```
PENDING（待签约）→ PENDING_ACTIVATION（已签约待激活）→ PAID（已支付）
                                                    ↓
                                               CANCELLED（审核失败）
```

```
                                               ▼
                    ┌─────────────────────────────────────────────┐
                    │              校验检查                         │

                                               ▼
```

**扣费计算逻辑**：

## 八、定时任务

### 8.1 任务列表

| 任务 | 执行时间 | Cron | 说明 |
|------|---------|------|------|
| 到期提醒 | 09:00 | 0 9 * * * | 体验期前3天/1天发送提醒 |
| 体验期到期扣费 | 10:00 | 0 10 * * * | 检查即将到期会员，触发扣费 |
| 扣费重试 | 10:30 | 30 10 * * * | 重试扣费失败的会员 |
| PENDING_CANCEL执行 | 11:00 | 0 11 * * * | 到期日执行取消 |
| 会员状态检查 | 12:00 | 0 12 * * * | 检查到期会员，更新状态 |

### 8.2 任务逻辑

#### 8.2.1 到期提醒

```
查询条件：
- 会员状态 = TRIAL
- 剩余天数 = 3 或 1

操作：
- 创建通知记录
- 发送短信/推送
```

#### 8.2.2 体验期到期扣费

```
查询条件：
- 会员状态 = TRIAL
- endDate = 今天

操作：
- 发送扣费指令给ETC记账系统
- 扣费成功：TRIAL → ACTIVE，通知粤运
- 扣费失败：记录失败日志
```

#### 8.2.3 扣费重试

```
查询条件：
- BillingRecord.status = FAILED
- createdAt >= 3天内

操作：
- 重新发送扣费指令
- 更新扣费状态
- 连续3天失败：自动取消会员
```

#### 8.2.4 PENDING_CANCEL执行

```
查询条件：
- 会员状态 = PENDING_CANCEL
- endDate = 今天

操作：
- 通知互联网系统解除签约
- 取消权益（状态变为EXPIRED）
- 会员状态变为CANCELLED
```

#### 8.2.5 会员状态检查

```
查询条件：
- 会员状态 = ACTIVE 或 TRIAL
- endDate < 今天

操作：
- 状态变为EXPIRED
- 权益状态变为EXPIRED
```

## 九、异常处理

### 9.1 业务异常

| 场景 | 处理方式 | 用户提示 |
|------|---------|---------|
| 签约失败 | 订单取消 | "签约失败，请重新尝试" |
| 审核失败 | 订单取消 | "审核未通过，原因：xxx" |
| 扣费失败 | 记录日志，每日重试 | "扣费失败，将在次日重试" |
| 连续3天扣费失败 | 自动取消会员 | "扣费连续失败，会员已取消" |
| 权益分配失败（总量限制） | 跳过该权益 | 不影响其他权益 |
| 一车一会员校验失败 | 拒绝激活 | "该车牌已有生效中的会员" |

### 9.2 系统异常

| 场景 | 处理方式 |
|------|---------|
| 粤运通知失败 | 记录日志，不影响主流程，后续补偿 |
| ETC记账系统通知失败 | 记录日志，重试机制 |
| 综合服务系统通知失败 | 记录日志，不影响主流程 |

### 9.3 补偿机制

| 场景 | 补偿方式 |
|------|---------|
| 扣费成功但会员创建失败 | 回滚扣费或手动创建会员 |
| 通知外部系统失败 | 定时重试 + 人工介入 |
