# 核心业务流程

> 基于系统交互说明整理
> 更新日期：2026-04-16

***

## 一、流程总览

| 流程           | 触发方       | 核心系统         | 涉及外部系统     |
| ------------ | --------- | ------------ | ---------- |
| ETC申办        | 用户        | 互联网系统        | 签约渠道       |
| 会员购买（ETC激活后） | 用户        | 营销系统、互联网系统   | 签约渠道、粤运系统  |
| 免费期到期扣费      | 系统（定时）    | 营销系统、ETC记账系统 | 签约渠道、粤运系统  |
| 次卡通行扣费       | ETC记账系统   | 营销系统         | 签约渠道       |
| 会员取消         | 用户/管理员/系统 | 营销系统         | 粤运系统、互联网系统 |
| 权益核销         | 粤运系统      | 营销系统         | -          |
| ETC设备注销      | 用户        | 互联网系统、营销系统   | 粤运系统       |
| 只换不修         | 用户        | 综合服务系统       | -          |

## 二、核心流程详解

### 2.1 ETC申办流程

**前置条件**：用户未办理ETC

```
用户 → 互联网系统 → 签约渠道

1. 用户在互联网系统发起ETC申办
2. 互联网系统调用签约渠道完成通行费签约
3. 签约渠道返回签约结果
4. ETC申办成功，等待卡签寄达

状态：订单待激活
```

**关键点**：

- 此阶段仅完成通行费签约，不涉及会员
- 签约渠道：支付宝/微信/银联

***

### 2.2 会员购买流程（ETC激活后）

**前置条件**：用户已收到ETC卡签，完成激活

```
用户 → 互联网系统 → 营销系统 → 互联网系统 → 签约渠道
       → ETC记账系统 → 综合服务系统 → 粤运系统

1. 用户收到ETC卡签，在互联网系统进行激活
2. 互联网系统通知营销系统：ETC激活成功
3. 营销系统展示激活成功页 + 会员产品推荐
4. 用户选择会员产品 → 同意协议
5. 互联网系统调用营销系统发起会员签约
6. 互联网系统调用签约渠道完成会员签约
7. 签约渠道返回签约结果 → 互联网系统 → 营销系统
8. 签约成功后进入审核流程
   - 审核通过：继续下一步
   - 审核失败：订单取消，通知用户
9. 营销系统发送扣费指令给ETC记账系统
   - 年卡：一次性扣费 ¥138
   - 月卡：扣首月 ¥16.8
   - 次卡：不扣费
10. ETC记账系统通过签约渠道执行扣费
11. 扣费结果返回 → ETC记账系统 → 营销系统
12. 扣费成功 → 营销系统创建会员（TRIAL，61天免费期）→ 分配权益
13. 营销系统通知：
    - 综合服务系统：只换不修权益开通
    - ETC记账系统：会员信息同步
    - 粤运系统：开通会员免费期

状态：订单待激活 → 已支付 → 会员 TRIAL
```

**关键点**：

- **一车一会员校验**：步骤3校验车牌是否已有生效会员
- **二次签约**：通行费签约（ETC申办时）+ 会员签约
- **审核环节**：签约成功后需审核，审核失败则订单取消
- **扣费差异**：年卡/月卡立即扣费，次卡按次扣费

***

### 2.3 免费期到期扣费流程

**前置条件**：会员处于 TRIAL 状态，即将满61天

```
营销系统（定时） → ETC记账系统 → 签约渠道 → 粤运系统

1. 到期前3天/1天，营销系统发送提醒通知
2. 免费期结束，营销系统发送扣费指令给ETC记账系统
   - 年卡：一次性扣费 ¥138（续费一年）
   - 月卡：扣 ¥16.8（续费一个月）
   - 次卡：体验期结束后按次扣费（体验期内不扣通行费）
3. ETC记账系统通过签约渠道执行扣费
4. 扣费结果返回 → ETC记账系统 → 营销系统
5. 扣费成功：
   - 会员状态 TRIAL → ACTIVE
   - 取消免费期权益，分配正式权益
   - 通知粤运系统：免费期结束，会员转正式
6. 扣费失败：
   - 记录失败日志
   - 每日轮扣重试
   - 连续3天扣费失败 → 自动取消会员
   - 通知粤运系统（BILLING_FAILED_CANCEL）
   - 通知ETC记账系统（会员取消）

状态：会员 TRIAL → ACTIVE 或 PENDING_CANCEL → CANCELLED
```

**关键点**：

- **到期提醒**：前3天、前1天两次提醒
- **轮扣重试**：每日重试，连续3天失败自动取消
- **次卡体验期**：体验期内不扣通行费，体验期结束后按次扣费

***

### 2.4 次卡通行扣费流程

**前置条件**：次卡会员，用户驾驶车辆通行高速

```
ETC记账系统 → 营销系统 → ETC记账系统 → 签约渠道

1. ETC记账系统发送通行扣费指令给营销系统
2. 营销系统校验：
   - 会员状态（TRIAL/ACTIVE）
   - 产品上架状态
   - 权益启用状态
3. 营销系统检查月封顶（¥20/月）
   - 未封顶：正常扣费 ¥1/次
   - 已封顶：扣费 ¥0 或差额
   - 封顶后：状态返回 CAPPED
4. 营销系统返回扣费指令给ETC记账系统
5. ETC记账系统通过签约渠道执行扣费
6. 扣费结果返回 → ETC记账系统

状态：扣费记录状态 SUCCESS/CAPPED
```

**关键点**：

- **主动权在ETC记账系统**：营销系统被动接收扣费指令
- **月封顶校验**：¥20/月封顶
- **状态判断**：校验产品状态 + 会员状态

***

### 2.5 会员取消流程

**前置条件**：会员处于 TRIAL/ACTIVE 状态

```
用户/管理员/扣费引擎 → 营销系统 → 粤运系统/ETC记账系统/互联网系统/综合服务系统

触发来源：
- USER_CANCEL：用户主动取消（前端5秒倒计时确认）
- ADMIN_CANCEL：后台管理员取消
- BILLING_FAILED：扣费引擎检测连续3天扣费失败
- ETC_CANCELLED：ETC设备注销

流程：
1. 营销系统更新会员状态为 PENDING_CANCEL
2. 营销系统通知粤运系统（MEMBER_CANCELLED）
3. 营销系统通知ETC记账系统（会员取消）
4. 营销系统通知综合服务系统（取消只换不修权益）
5. 权益保留至到期日
6. 到期日：
   - 营销系统通知互联网系统解除会员签约
   - 营销系统取消权益
   - 会员状态变为 CANCELLED

状态：会员 TRIAL/ACTIVE → PENDING_CANCEL → CANCELLED
```

**关键点**：

- **待取消状态**：PENDING\_CANCEL 期间权益保留
- **多方通知**：粤运、ETC记账、综合服务、互联网系统
- **解约时机**：到期日才真正解约

***

### 2.6 权益核销流程

**前置条件**：用户享有粤运拯救权益，使用救援服务

```
粤运系统 → 营销系统

1. 用户拨打粤运救援电话
2. 粤运系统提供救援服务
3. 粤运系统通知营销系统：粤运拯救服务已使用
4. 营销系统核销对应权益（usedCount + 1）
5. 若 usedCount >= totalCount，权益状态变为 USED

状态：权益 ACTIVE → USED（次数用完时）
```

**关键点**：

- **核销主动权在粤运**：营销系统被动接收核销通知
- **次数管理**：支持多次权益

***

### 2.7 ETC设备注销流程

**前置条件**：用户主动注销ETC设备

```
用户 → 互联网系统 → 营销系统 → 粤运系统/ETC记账系统/综合服务系统

1. 用户在互联网系统申请ETC设备注销
2. 互联网系统通知营销系统：ETC设备注销
3. 营销系统自动取消会员（cancelReason: ETC_CANCELLED）
4. 营销系统通知：
   - 粤运系统
   - ETC记账系统
   - 综合服务系统（取消只换不修权益）
5. 权益立即失效

状态：会员 → CANCELLED
```

**关键点**：

- **立即取消**：无需 PENDING\_CANCEL 过渡
- **权益立即失效**：不保留至到期日

***

### 2.8 只换不修流程

**前置条件**：会员享有只换不修权益，ETC设备故障

```
用户 → 综合服务系统

1. 用户ETC设备故障，申请更换
2. 综合服务系统校验维保期
   - 维保期内：免费更换设备
   - 维保期外：用户自行承担费用
3. 完成设备更换

触发时机：
- 会员开通时：营销系统通知综合服务系统开通权益
- 会员取消时：营销系统通知综合服务系统取消权益
```

**关键点**：

- **维保期管理**：综合服务系统负责
- **独立系统**：与营销系统解耦

## 三、状态机

### 3.1 订单状态

```
PENDING（待支付）
    ↓ 签约成功
PENDING_ACTIVATION（待激活）
    ↓ 审核通过
PAID（已支付）
    ↓ 激活成功
COMPLETED（已完成）

分支：
- 审核失败 → CANCELLED
- 用户取消 → CANCELLED
- 退款 → REFUNDED
```

### 3.2 会员状态

```
TRIAL（体验期，61天）
    │
    ├─ 扣费成功 ─→ ACTIVE（正式会员）
    │                  │
    │                  ├─ 到期未续费 ─→ EXPIRED
    │                  │
    │                  └─ 申请取消 ─→ PENDING_CANCEL ─→ CANCELLED
    │
    └─ 申请取消 ─→ PENDING_CANCEL ─→ CANCELLED

特殊路径：
- 扣费连续3天失败 ─→ PENDING_CANCEL ─→ CANCELLED
- ETC注销 ─→ CANCELLED（立即，无过渡）
```

### 3.3 权益状态

```
ACTIVE（可用）
    │
    ├─ 次数用完 ─→ USED
    │
    ├─ 过期 ─→ EXPIRED
    │
    └─ 会员取消 ─→ EXPIRED
```

## 四、定时任务

| 任务                   | 频率 | 说明            |
| -------------------- | -- | ------------- |
| 到期提醒                 | 每日 | 体验期前3天/1天发送提醒 |
| 体验期到期扣费              | 每日 | 检查即将到期会员，触发扣费 |
| 轮扣重试                 | 每日 | 重试扣费失败的会员     |
| 会员状态检查               | 每日 | 检查到期会员，更新状态   |
| PENDING\_CANCEL转正式取消 | 每日 | 到期日执行取消       |

***

## 五、异常处理

| 场景           | 处理方式                 |
| ------------ | -------------------- |
| 签约失败         | 订单取消，通知用户重新签约        |
| 审核失败         | 订单取消，通知用户            |
| 扣费失败         | 记录日志，每日重试，连续3天失败自动取消 |
| 粤运通知失败       | 记录日志，不影响主流程，后续补偿     |
| 权益分配失败（总量限制） | 跳过该权益，记录日志，继续处理其他权益  |

## 六、数据一致性

### 6.1 扣费结果处理

```
扣费成功：
1. 创建会员记录（TRIAL）
2. 分配权益
3. 更新订单状态（PAID）
4. 通知外部系统

扣费失败：
1. 记录扣费日志
2. 设置重试标记
3. 连续3天失败 → 取消会员
```

### 6.2 会员取消处理

```
PENDING_CANCEL 状态：
1. 权益保留至到期日
2. 到期日执行：
   - 解除签约
   - 取消权益
   - 状态转 CANCELLED
   - 通知外部系统
```
