## 一、项目概述

### 1.1 项目背景

粤通卡ETC会员营销平台，为ETC用户提供会员服务（年卡/月卡/次卡），用户在ETC申办时体验会员，收到ETC产品激活后自动扣费开通。平台与粤运系统对接，实现权益服务的闭环管理。

### 1.2 项目愿景

搭建完整业务闭环，支持真实的增删改查操作，现代SaaS风格界面，实现会员全生命周期管理。

### 1.3 范围界定

**项目范围（In Scope）**
- 用户端：会员购买、权益查看、扣费记录、取消退款
- 后台端：仪表盘、权益管理、订单管理、会员管理、结算对账、系统配置
- 外部系统对接：粤运系统、ETC记账系统、互联网系统、综合服务系统

**项目范围外（Out of Scope）**
- 权益包组合购买
- 权益转赠功能

## 三、核心业务规则

### 3.1 激活扣费

**申办时不扣费，ETC激活时才扣费。**

1. 用户在ETC发行时选择签约渠道和会员产品，仅创建订单（不扣费、不开通会员）
2. 用户收到ETC产品后进行激活，此时通过签约渠道扣费
   - **年卡**：一次性扣费 ¥138
   - **月卡**：扣首月 ¥16.8
   - **次卡**：不扣费（按次扣费模式）
3. 扣费成功：开通会员（61天免费体验期）→ 分配权益 → 通知粤运 → 通知ETC记账系统
4. 扣费失败：不创建会员，记录失败，每日重试

### 3.2 二次签约

用户需要完成两次签约：

| 签约类型 | 触发时机 | 签约内容 |
|---------|---------|---------|
| 通行费签约 | ETC申办时 | 授权从签约渠道扣取通行费 |
| 会员签约 | 购买会员时 | 授权从签约渠道扣取会员费 |

### 3.3 一车一会员

每辆车只能购买一个会员产品。

- 激活时校验车牌是否已有生效中的会员（TRIAL/ACTIVE/PENDING_CANCEL）
- 有则拒绝激活，提示"该车牌已有生效中的会员"

### 3.4 免费体验期

新会员开通后享受 **61天** 免费体验期（状态为 TRIAL）。

- 体验期内可随时取消，不产生费用
- 到期前3天、1天发送提醒通知
- 体验期满未取消，系统自动从签约渠道扣取会员费
- 扣费成功后状态变为 ACTIVE

### 3.5 会员产品类型

| 类型 | 说明 | 扣费方式 | 权益 |
|------|------|---------|------|
| 年卡（YEARLY） | 年度会员 | 一次性扣费 ¥138/年 | 粤运拯救、只换不修、中石化年卡权益 |
| 月卡（MONTHLY） | 月度会员 | 每月扣费 ¥16.8/月 | 粤运拯救（限500元） |
| 次卡（PER_USE） | 按次计费 | 每次通行扣费 ¥1/次，月封顶 ¥20 | 粤运拯救、高速意外险 |

## 四、核心业务流程

### 4.1 ETC申办流程

**前置条件**：用户未办理ETC

```
用户 → 互联网系统 → 签约渠道

1. 用户在互联网系统发起ETC申办
2. 互联网系统调用签约渠道完成通行费签约
3. 签约渠道返回签约结果
4. ETC申办成功，等待卡签寄达

状态：订单待激活
```

### 4.2 会员购买流程（ETC激活后）

**前置条件**：用户已收到ETC卡签，完成激活

```
用户 → 互联网系统 → 营销系统 → 互联网系统 → 签约渠道
       → ETC记账系统 → 综合服务系统 → 粤运系统

1. 用户收到ETC卡签，在互联网系统进行激活
2. 互联网系统通知营销系统：ETC激活成功
3. 营销系统展示激活成功页 + 会员产品推荐
4. 用户选择会员产品 → 同意协议
5. 营销系统调用互联网系统发起会员签约
6. 互联网系统调用签约渠道完成会员签约
7. 签约渠道返回签约结果 → 互联网系统 → 营销系统
8. 签约成功后进入审核流程
   - 审核通过：继续下一步
   - 审核失败：订单取消，通知用户
9. 营销系统发送扣费指令给ETC记账系统
10. ETC记账系统通过签约渠道执行扣费
11. 扣费结果返回 → ETC记账系统 → 营销系统
12. 扣费成功 → 营销系统创建会员（TRIAL，61天免费期）→ 分配权益
13. 营销系统通知：
    - 综合服务系统：只换不修权益开通
    - ETC记账系统：会员信息同步
    - 粤运系统：开通会员免费期

状态：订单 PENDING_ACTIVATION → PAID，会员 TRIAL
```

### 4.3 免费期到期扣费流程

**前置条件**：会员处于 TRIAL 状态，即将满61天

```
营销系统（定时） → ETC记账系统 → 签约渠道 → 粤运系统

1. 到期前3天/1天，营销系统发送提醒通知
2. 免费期结束，营销系统发送扣费指令给ETC记账系统
   - 年卡：一次性扣费 ¥138（续费一年）
   - 月卡：扣 ¥16.8（续费一个月）
   - 次卡：无此环节（按次扣费，无免费期概念）
3. ETC记账系统通过签约渠道执行扣费
4. 扣费结果返回 → ETC记账系统 → 营销系统
5. 扣费成功：
   - 会员状态 TRIAL → ACTIVE
   - 取消免费期权益，分配正式权益
   - 通知粤运系统：免费期结束，会员转正式
6. 扣费失败：
   - 记录失败日志
   - 每日轮扣重试
   - 连续3天扣费失败 → 自动取消会员
   - 通知粤运系统（BILLING_FAILED_CANCEL）
   - 通知ETC记账系统（会员取消）

状态：会员 TRIAL → ACTIVE 或 PENDING_CANCEL → CANCELLED
```

### 4.4 次卡通行扣费流程

**前置条件**：次卡会员，用户驾驶车辆通行高速

```
ETC记账系统 → 营销系统 → ETC记账系统 → 签约渠道

1. ETC记账系统发送通行扣费指令给营销系统
2. 营销系统校验：
   - 会员状态（TRIAL/ACTIVE）
   - 产品上架状态
   - 权益启用状态
3. 营销系统检查月封顶（¥20/月）
   - 未封顶：正常扣费 ¥1/次
   - 已封顶：扣费 ¥0 或差额
   - 封顶后：状态返回 CAPPED
4. 营销系统返回扣费指令给ETC记账系统
5. ETC记账系统通过签约渠道执行扣费
6. 扣费结果返回 → ETC记账系统

状态：扣费记录状态 SUCCESS/CAPPED
```

### 4.5 会员取消流程

**前置条件**：会员处于 TRIAL/ACTIVE 状态

```
用户/管理员/扣费引擎 → 营销系统 → 粤运系统/ETC记账系统/互联网系统/综合服务系统

触发来源：
- USER_CANCEL：用户主动取消（前端5秒倒计时确认）
- ADMIN_CANCEL：后台管理员取消
- BILLING_FAILED：扣费引擎检测连续3天扣费失败
- ETC_CANCELLED：ETC设备注销

流程：
1. 营销系统更新会员状态为 PENDING_CANCEL
2. 营销系统通知粤运系统（MEMBER_CANCELLED）
3. 营销系统通知ETC记账系统（会员取消）
4. 营销系统通知综合服务系统（取消只换不修权益）
5. 权益保留至到期日
6. 到期日：
   - 营销系统通知互联网系统解除会员签约
   - 营销系统取消权益
   - 会员状态变为 CANCELLED

状态：会员 TRIAL/ACTIVE → PENDING_CANCEL → CANCELLED
```

### 4.6 权益核销流程

**前置条件**：用户享有粤运拯救权益，使用救援服务

```
粤运系统 → 营销系统

1. 用户拨打粤运救援电话
2. 粤运系统提供救援服务
3. 粤运系统通知营销系统：粤运拯救服务已使用
4. 营销系统核销对应权益（usedCount + 1）
5. 若 usedCount >= totalCount，权益状态变为 USED

状态：权益 ACTIVE → USED（次数用完时）
```

### 4.7 ETC设备注销流程

**前置条件**：用户主动注销ETC设备

```
用户 → 互联网系统 → 营销系统 → 粤运系统/ETC记账系统/综合服务系统

1. 用户在互联网系统申请ETC设备注销
2. 互联网系统通知营销系统：ETC设备注销
3. 营销系统自动取消会员（cancelReason: ETC_CANCELLED）
4. 营销系统通知：
   - 粤运系统
   - ETC记账系统
   - 综合服务系统（取消只换不修权益）
5. 权益立即失效

状态：会员 → CANCELLED（立即，无过渡）
```

### 4.8 只换不修流程

**前置条件**：会员享有只换不修权益，ETC设备故障

```
用户 → 综合服务系统

1. 用户ETC设备故障，申请更换
2. 综合服务系统校验维保期
   - 维保期内：免费更换设备
   - 维保期外：用户自行承担费用
3. 完成设备更换

触发时机：
- 会员开通时：营销系统通知综合服务系统开通权益
- 会员取消时：营销系统通知综合服务系统取消权益
```

| isSigned | Boolean | 否 | 是否已签约，默认false |

| activatedAt | DateTime | 否 | 激活时间 |
| paidAt | DateTime | 否 | 支付时间 |

**订单状态流转：**

```
PENDING → PENDING_ACTIVATION → PAID → COMPLETED
                                ↓
                            CANCELLED / REFUNDED
```

| 状态 | 说明 |
|------|------|
| PENDING | 待签约 |
| PENDING_ACTIVATION | 已签约待激活（申办成功后的状态） |
| PAID | 已支付（激活扣费成功后） |
| COMPLETED | 已完成 |
| CANCELLED | 已取消（审核失败/用户取消） |
| REFUNDED | 已退款 |

### 5.6 Member（会员）

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| id | String | 自动 | 主键 |
| userId | String | 是 | 关联用户 |
| productId | String | 是 | 关联产品 |
| status | String | 否 | 会员状态，默认TRIAL |
| startDate | DateTime | 是 | 开始日期 |
| endDate | DateTime | 是 | 结束日期 |
| isTrial | Boolean | 否 | 是否体验期，默认true |
| cancelReason | String | 否 | 取消原因 |
| cancelAt | DateTime | 否 | 取消时间 |
| warrantyEndDate | DateTime | 否 | 维保到期日期 |
| plateNumber | String | 否 | 绑定车牌 |
| plateColor | String | 否 | 车牌颜色 |

**会员状态流转：**

```
TRIAL ──(体验期结束扣费成功)──→ ACTIVE
  │                              │
  │                              └──(申请取消)──→ PENDING_CANCEL ──(到期)──→ CANCELLED
  │
  └──(申请取消)──→ PENDING_CANCEL ──(到期)──→ CANCELLED

TRIAL/ACTIVE ──(到期未续费)──→ EXPIRED
TRIAL/ACTIVE ──(扣费连续3天失败)──→ PENDING_CANCEL
TRIAL/ACTIVE ──(ETC注销)──→ CANCELLED（立即）
```

| 状态 | 说明 |
|------|------|
| TRIAL | 体验期（61天） |
| ACTIVE | 正式会员 |
| PENDING_CANCEL | 待取消（权益保留至到期日） |
| CANCELLED | 已取消 |
| EXPIRED | 已过期 |

**取消原因：**

| 原因 | 说明 |
|------|------|
| USER_CANCEL | 用户主动取消（前端5秒倒计时确认） |
| ADMIN_CANCEL | 后台管理员取消 |
| BILLING_FAILED | 扣费引擎检测连续3天扣费失败自动取消 |
| ETC_CANCELLED | ETC设备注销 |

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|

| userId | String | 是 | 关联用户 |
| rightId | String | 是 | 关联权益 |
| memberId | String | 是 | 关联会员 |
| status | String | 否 | ACTIVE/USED/EXPIRED，默认ACTIVE |
| totalCount | Integer | 否 | 总次数，默认1 |

## 十、异常处理

| ETC记账系统通知失败 | 记录日志，重试机制 |
