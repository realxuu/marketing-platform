# 粤通卡ETC会员营销平台 - 业务需求文档

***

## 一、项目背景

粤通卡ETC会员营销平台，为ETC用户提供会员服务（年卡/月卡/次卡），用户在ETC申办时体验会员，收到ETC产品激活后自动扣费开通。平台与粤运系统对接，实现权益服务的闭环管理。

***

## 二、核心业务规则

### 2.1 购销分离

**申办时不扣费，ETC激活时才扣费。**

- 用户在ETC发行时选择签约渠道和会员产品，仅创建订单（不扣费、不开通会员）
- 用户收到ETC产品后进行激活，此时通过签约渠道扣费
- 扣费成功：开通会员（2个月免费体验期）→ 分配权益 → 通知粤运
- 扣费失败：不创建会员，记录失败

### 2.2 一车一会员

每辆车只能购买一个会员产品。激活时校验车牌是否已有生效中的会员（TRIAL/ACTIVE/PENDING\_CANCEL），有则拒绝激活。

### 2.3 免费体验期

新会员开通后享受61天免费体验期（状态为TRIAL），体验期内可随时取消不产生费用。体验期满未取消，系统自动从签约渠道扣取会员费，扣费成功后状态变为ACTIVE。

### 2.4 会员产品类型

| 类型           | 说明   | 扣费方式              |
| ------------ | ---- | ----------------- |
| 年卡（YEARLY）   | 年度会员 | 一次性扣费¥138/年       |
| 月卡（MONTHLY）  | 月度会员 | 每月扣费¥16.8/月       |
| 次卡（PER\_USE） | 按次计费 | 每次通行扣费¥1/次，月封顶¥20 |

### 4.5 Order（订单）

| 字段                | 类型       | 必填 | 说明                          |
| ----------------- | -------- | -- | --------------------------- |
| id                | String   | 自动 | 主键                          |
| userId            | String   | 是  | 关联用户                        |
| productId         | String   | 是  | 关联产品                        |
| amount            | Float    | 是  | 订单金额                        |
| status            | String   | 否  | 订单状态，默认PENDING              |
| payMethod         | String   | 否  | 支付方式：WECHAT/ALIPAY/UNIONPAY |
| channel           | String   | 否  | 签约渠道：WECHAT/ALIPAY/UNIONPAY |
| isActivated       | Boolean  | 否  | 是否已激活，默认false               |
| activatedAt       | DateTime | 否  | 激活时间                        |
| agreementAccepted | Boolean  | 否  | 是否同意协议，默认false              |
| paidAt            | DateTime | 否  | 支付时间                        |

**订单状态流转：**

```
PENDING → PENDING_ACTIVATION → PAID
                                ↓
                            CANCELLED / REFUNDED
```

- PENDING：待支付
- PENDING\_ACTIVATION：待激活（申办成功后的状态，尚未扣费）
- PAID：已支付（激活扣费成功后）
- CANCELLED：已取消
- REFUNDED：已退款

### 4.6 Member（会员）

| 字段              | 类型       | 必填 | 说明           |
| --------------- | -------- | -- | ------------ |
| id              | String   | 自动 | 主键           |
| userId          | String   | 是  | 关联用户         |
| productId       | String   | 是  | 关联产品         |
| status          | String   | 否  | 会员状态，默认TRIAL |
| startDate       | DateTime | 是  | 开始日期         |
| endDate         | DateTime | 是  | 结束日期         |
| isTrial         | Boolean  | 否  | 是否体验期，默认true |
| cancelReason    | String   | 否  | 取消原因         |
| cancelAt        | DateTime | 否  | 取消时间         |
| warrantyEndDate | DateTime | 否  | 维保到期日期       |
| plateNumber     | String   | 否  | 绑定车牌         |
| plateColor      | String   | 否  | 车牌颜色         |

**会员状态流转：**

```
TRIAL ──(体验期结束扣费成功)──→ ACTIVE
  │                              │
  │                              └──(申请取消)──→ PENDING_CANCEL ──(到期)──→ CANCELLED
  │
  └──(申请取消)──→ PENDING_CANCEL ──(到期)──→ CANCELLED

TRIAL/ACTIVE ──(到期未续费)──→ EXPIRED
TRIAL/ACTIVE ──(扣费连续3天失败)──→ PENDING_CANCEL
```

**取消原因：**

| 原因              | 说明                 |
| --------------- | ------------------ |
| USER\_CANCEL    | 用户主动取消（前端5秒倒计时确认）  |
| ADMIN\_CANCEL   | 后台管理员取消            |
| BILLING\_FAILED | 扣费引擎检测连续3天扣费失败自动取消 |
| ETC\_CANCELLED  | ETC设备注销            |

## 五、API接口

### 5.1 用户模块

#### GET /api/users

查询所有用户，包含会员和产品信息。

#### POST /api/users

创建用户。

```json
{ "phone": "13800138001", "name": "张三", "plateNumber": "粤A12345", "plateColor": "BLUE", "idNumber": "440101199001011234" }
```

### 5.2 产品模块

#### GET /api/products

查询所有会员产品，包含关联权益。

### 5.5 激活模块

#### POST /api/activate

ETC激活，扣费并开通会员。

```json
{ "orderId": "订单ID" }
```

**处理步骤：**

1. 查询订单（含用户和产品权益信息）
2. 校验订单未激活
3. 从用户信息获取车牌号和车牌颜色
4. 校验车牌唯一性（一车一会员）
5. 创建会员记录（TRIAL，61天体验期）
6. 分配用户权益（检查总量限制，达到上限的权益跳过）
7. 更新订单状态为PAID，isActivated=true
8. 通知粤运（action: MEMBER\_ACTIVATED）

2. 校验会员未被取消
3. 更新会员状态为PENDING\_CANCEL，记录取消原因和时间
4. 创建取消确认通知
5. 通知粤运（action: MEMBER\_CANCELLED）
