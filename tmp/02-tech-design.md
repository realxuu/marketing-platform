# 粤通卡ETC营销平台 - 技术方案

## 一、技术架构

### 技术栈
- **前端框架**：Next.js 16 + React 19
- **UI组件**：Tailwind CSS + shadcn/ui
- **数据库**：Prisma ORM + PostgreSQL（线上）/ SQLite（本地开发）
- **状态管理**：React Query + Zustand
- **部署**：Vercel

## 二、数据模型

### 核心实体关系

- **User** → 用户，关联 Member 和 Order
- **MemberProduct** → 会员产品（年卡/月卡/次卡），关联 ProductRight
- **Right** → 权益定义，关联 ProductRight
- **ProductRight** → 产品-权益关联表
- **Order** → 订单，关联 User 和 MemberProduct
- **Member** → 会员记录，关联 User、MemberProduct、BillingRecord
- **BillingRecord** → 扣费记录
- **Settlement** → 结算记录
- **UserRight** → 权益发放记录，关联 RightUsage
- **RightUsage** → 权益使用/核销记录
- **SystemConfig** → 系统参数配置
- **BillingLog** → 扣费日志

## 三、核心业务流程

### 1. 购买流程
```
用户选择产品 → 确认订单 → 模拟支付 → 支付成功 → 创建会员记录
```

### 2. 扣费流程
```
后台手动触发 → 创建扣费记录 → 展示扣费明细
```

### 3. 退款流程
```
用户申请/后台操作 → 订单状态变更为退款 → 展示退款记录
```
